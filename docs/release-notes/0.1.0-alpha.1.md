# OpenEdifier 0.1.0-alpha.1

OpenEdifier 的首个公开 alpha，提供经过验证的 S260 本地控制能力、CLI 和原生 macOS App。

## 支持边界

- 硬件：EDIFIER S260；
- 已验证固件：`01.00.00`；
- 网络：可信局域网；
- macOS：Apple Silicon、macOS 26；
- 能力：发现、状态、输入源、音量、EQ、播放控制和实时事件。

其他型号、固件和平台尚未验证。恢复出厂、固件升级、关机和重命名等破坏性能力不在公开 API 中。

## CLI 安装

```bash
brew install xuhuanxxx/tap/open-edifier
edifier --version
```

## macOS 安装

下载 `OpenEdifier-0.1.0-alpha.1-macos-arm64.dmg` 及其 `.sha256`，校验后将 App 拖入 `/Applications`。

该 App 使用 ad-hoc 签名，没有 Developer ID，也没有经过 Apple 公证。确认下载来源和 checksum 后，首次尝试打开，再前往“系统设置 → 隐私与安全性”选择“仍要打开”。不要全局关闭 Gatekeeper。

## 已知限制

- 自动发现依赖 S260 的 AirPlay mDNS 广播；
- 控制协议没有认证和加密，只能在可信局域网使用；
- 播放命令只能确认设备收到命令，实际效果取决于当前输入源；
- macOS App 当前每 5 秒刷新状态，尚未接入实时事件流；
- 只提供 arm64 App，不支持 Intel Mac。

## 自动验证

release workflow 会在创建本 prerelease 前执行 Rust 格式、clippy、测试、文档、workspace package、macOS 构建、ad-hoc 签名和 checksum 验证。

## 安装验证

- public Homebrew tap 已通过 strict online audit、直接安装、formula test、版本/帮助、upgrade 幂等检查和卸载；
- GitHub Release 的 DMG/checksum 已通过匿名下载、SHA-256、image CRC、bundle 结构、签名和版本回读；
- 带 quarantine 的非开发 macOS 用户已按“系统设置 → 隐私与安全性 → 仍要打开”流程确认 App 可启动。

## 实机验证记录

2026-07-19 使用固件 `01.00.00` 的 S260 对当前发布候选执行：

- mDNS 发现和完整状态读取；
- 音量 `8 → 9 → 8`，每次写入后回读验证；
- EQ `2 → 1 → 2`，每次写入后回读验证；
- 在当前可用的 AirPlay 输入源上重设 AirPlay 并回读验证；
- 当前播放状态下发送 `play`，设备返回 ACK；
- 建立实时事件流，音量 `8 → 9` 时收到 `volume` 事件，随后恢复到 `8`。

未切换到不可用输入源，未执行恢复出厂、固件升级、关机、重命名或网络破坏性测试。最终状态恢复为 AirPlay、音量 `8`、EQ `2`、播放中。
