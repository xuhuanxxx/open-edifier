# OpenEdifier 0.2.0-alpha.1

> 候选状态：尚未创建 tag 或 GitHub prerelease。

本版本是不兼容的运行时边界修复，重点处理音量写后验证、macOS 轮询竞争、型号识别、事件断线忙等、畸形响应恢复和隐私安全错误，并新增面向 Home Assistant 的纯 Python 异步客户端源码包。

## 不兼容变化

- `ClientConfig.timeout` 拆分为连接、请求、验证窗口和验证间隔；
- `Client` 的状态和修改方法直接返回公共 `DeviceStatus`，不再公开 `SpeakerStatus`；
- `DeviceEvents::next_event` 必须接收调用方最大等待时间；
- facade `discover` 只返回当前 build 支持的设备，低层候选发现改为 `discover_candidates`；
- S260 framed JSON decoder 不再公开；
- Swift bridge 的 `error` 从字符串改为结构化对象；
- 不提供旧 Rust API、事件签名或 bridge JSON 的兼容层。

## 修复

- 输入源、音量和 EQ 共用有截止时间的写后验证循环，允许设备状态短暂延迟；
- macOS 串行执行设备操作，滑块编辑期间暂停轮询，失败时回滚到最后确认音量；
- S260 只根据硬件型号 `EDF100122` 选择驱动，不信任可修改的设备名称；
- 事件重连在调用预算内等待并保留最后错误，不再让 CLI 退避期间忙等；
- 畸形 JSON 帧不会永久污染连接；
- 拒绝错误不会携带完整设备响应，bridge 提供稳定错误 kind 和白名单上下文；
- 状态和事件增加音量、EQ 与已知载荷不变量校验。

## Python 客户端

- `bindings/python` 可构建为 `open_edifier-0.2.0a1-py3-none-any.whl`，无运行时依赖；
- 原生 `asyncio` API 提供状态、输入源、音量、EQ、播放和可重连事件流；
- 不调用 CLI、Rust 动态库或其他本地产物；设备发现留给 Home Assistant Zeroconf；
- 当前只支持源码安装，尚未发布 PyPI，Home Assistant integration 仍在规划中。
