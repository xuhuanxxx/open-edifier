# AA EC/BB EC 二进制协议

本文记录了固件 `01.00.00` 的 S260 实机经过脱敏的观察结果。其他 EDIFIER 产品也被报告使用同族协议，但没有对应型号的证据前，OpenEdifier 不会假设命令兼容。

## 帧格式

```text
请求 = AA EC | command u16-le | payload_length u8 | payload | checksum u8
响应 = BB EC | command u16-le | payload_length u8 | payload | checksum u8
校验 = 前面所有帧字节的 wrapping sum
```

长度字段为一字节。早期探索脚本曾错误地按两字节解析；捕获到的控制响应和最终可复用实现共同确认了一字节格式。

模组大约每隔数秒主动发送一次命令 `0x003F` 的心跳。连接即使处于空闲状态，设备状态变化也可能以未经请求的 `BB EC` 帧到达。

## 已验证的 S260 事件与命令

| 命令 | 含义 | 载荷 |
|---:|---|---|
| `0x0061` | 输入源状态 | `1E N`；N 为 1 蓝牙、2 AUX、3 USB、4 AirPlay |
| `0x0062` | 设置输入源 | 请求 `1E N`；新状态通过 `0x0061` 上报 |
| `0x0066` | 音量状态 | `最大值 当前值` |
| `0x0067` | 设置音量 | 请求包含一个音量字节；新状态通过 `0x0066` 上报 |
| `0x0068` | 播放状态 | 0 停止、1 播放、2 暂停 |
| `0x00C2` | 播放操作 | 0 播放、1 暂停、4 下一首、5 上一首 |
| `0x00C4` | 设置 EQ 预设 | 一个预设编号字节 |
| `0x00D5` | EQ 预设状态 | 一个预设编号字节 |
| `0x0050` | 曲目信息 | 尚未解码 |
| `0x00D8` | 能力位图 | 32 字节，尚未解码 |

S260 经常只响应一条连接上的第一个二进制请求，而且可能丢失响应。OpenEdifier 当前使用 framed JSON 发送命令，二进制通道只承载事件，从而避免依赖需要大量重试的二进制命令行为。

校验和有效但当前未知的事件命令不会被丢弃，而是通过 `DeviceEvent::Unknown` 向上暴露。

增量 decoder 会跳过校验和无效的候选帧，并在下一个响应 magic 处重新同步。已经建立的 socket 关闭后，S260 事件流会重置 decoder 状态，并采用有界指数退避重连；调用方仍会周期性收到 `None`，以便应用及时响应取消操作。
